# 学生管理系统前端规格说明书

## 1. 项目概述

### 项目名称
学生管理系统 (Student Management System)

### 项目类型
Web前端单页应用 (SPA)

### 核心功能简述
一个完整的学生信息管理平台，提供学生的增删改查、搜索筛选、数据统计等功能，采用现代化的用户界面设计。

### 目标用户
- 学校教务管理人员
- 班主任和任课教师
- 学生管理人员

---

## 2. 技术栈

- **前端框架**: React 18
- **UI组件库**: Ant Design 5.x
- **样式方案**: CSS3 + Ant Design内置样式
- **状态管理**: React Hooks (useState, useEffect)
- **图标库**: Ant Design Icons
- **数据存储**: LocalStorage (浏览器本地存储)
- **构建工具**: Vite

---

## 3. 功能需求

### 3.1 学生列表展示
- 表格形式展示所有学生信息
- 每行显示：学号、姓名、性别、年龄、班级、联系电话、操作按钮
- 支持分页显示（每页10条数据）
- 支持按学号或姓名排序

### 3.2 添加学生
- 点击"添加学生"按钮打开模态框
- 表单字段：
  - 学号（必填，唯一性验证）
  - 姓名（必填）
  - 性别（必填，单选）
  - 年龄（必填，18-25岁范围验证）
  - 班级（必填）
  - 联系电话（必填，手机号格式验证）
- 提交前进行表单验证
- 添加成功后自动刷新列表

### 3.3 编辑学生
- 点击"编辑"按钮打开预填充的编辑模态框
- 表单字段与添加学生相同
- 学号字段设置为只读
- 更新成功后自动刷新列表

### 3.4 删除学生
- 点击"删除"按钮显示确认对话框
- 确认后删除学生记录
- 删除成功后自动刷新列表

### 3.5 搜索功能
- 搜索框支持按姓名或学号模糊搜索
- 实时搜索（输入时自动搜索）
- 搜索结果高亮显示匹配内容

### 3.6 筛选功能
- 按班级下拉筛选
- 按性别下拉筛选
- 支持组合筛选

### 3.7 数据统计
- 卡片展示统计信息：
  - 学生总人数
  - 各班级人数分布
  - 男女比例

## 4. UI/UX设计规范

### 4.1 整体布局
- **顶部导航栏**: 系统标题、操作按钮（添加学生）
- **统计卡片区**: 数据统计信息展示
- **搜索筛选区**: 搜索框、筛选下拉框
- **数据表格区**: 学生信息表格、分页控件
- **底部**: 版权信息

### 4.2 色彩方案
- **主色调**: #1890ff (Ant Design 蓝色)
- **背景色**: #f0f2f5 (浅灰背景)
- **卡片背景**: #ffffff (白色)
- **文字主色**: #262626 (深灰)
- **文字次色**: #8c8c8c (浅灰)
- **成功色**: #52c41a (绿色)
- **警告色**: #faad14 (橙色)
- **错误色**: #ff4d4f (红色)

### 4.3 字体规范
- **标题字体**: "Helvetica Neue", Arial, sans-serif
- **正文字体**: "Helvetica Neue", Arial, sans-serif
- **字号梯度**:
  - 页面标题: 24px
  - 卡片标题: 16px
  - 表格正文: 14px
  - 辅助文字: 12px

### 4.4 间距系统
- **页面边距**: 24px
- **卡片间距**: 16px
- **元素间距**: 8px
- **内边距**: 卡片内容区域 24px

### 4.5 响应式断点
- **桌面端**: ≥1200px (完整布局)
- **平板端**: 768px-1199px (表格横向滚动)
- **移动端**: <768px (单列布局，简化表格)

## 5. 数据模型

### 5.1 学生信息 (Student)
```typescript
interface Student {
  id: string;          // 唯一标识符
  studentId: string;   // 学号
  name: string;        // 姓名
  gender: '男' | '女'; // 性别
  age: number;         // 年龄
  className: string;   // 班级
  phone: string;       // 联系电话
  createdAt: string;   // 创建时间
  updatedAt: string;   // 更新时间
}
```

### 5.2 初始测试数据
系统应包含10条初始学生数据用于演示，数据涵盖不同班级和性别。

## 6. 表单验证规则

| 字段 | 必填 | 验证规则 | 错误提示 |
|------|------|----------|----------|
| 学号 | 是 | 非空，4-10位数字或字母组合，唯一性检查 | 请输入4-10位学号，该学号已存在 |
| 姓名 | 是 | 非空，2-20个字符 | 请输入姓名 |
| 性别 | 是 | 单选必选 | 请选择性别 |
| 年龄 | 是 | 18-25岁整数 | 年龄必须在18-25岁之间 |
| 班级 | 是 | 非空 | 请输入班级 |
| 联系电话 | 是 | 11位手机号格式 | 请输入正确的11位手机号 |

## 9. 异常处理

### 9.1 数据异常
- 学号重复：提示"该学号已存在，请使用其他学号"
- 数据格式错误：显示具体的字段错误信息
- 数据为空：显示"暂无学生信息，请添加"

### 9.2 操作异常
- 添加失败：提示"添加失败，请重试"
- 修改失败：提示"修改失败，请重试"
- 删除失败：提示"删除失败，请重试"

### 9.3 存储异常
- LocalStorage已满：提示"存储空间不足，请清理浏览器缓存"
- 数据损坏：自动重置为初始数据

---

## 10. 备注

- 所有数据存储在浏览器LocalStorage中，页面刷新后数据保留
- 初始加载时会检查LocalStorage是否有数据，无则初始化10条演示数据
- 学号作为唯一标识，不允许重复修改
